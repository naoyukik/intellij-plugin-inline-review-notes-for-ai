# Initial Concept

IntelliJ の PR コードレビュー UI と同じ操作感で、ローカルファイルにインラインコメントを付与する。コメントは JSON 形式で保存され、AI がそのファイルを読むことで、ユーザーの指摘を即座に把握できるようにする。

## 目的
- コードを読みながら、AI への指摘事項をその場で記録できるようにする
- 口頭説明やコピペをなくし、レビューの往復を短縮する
- AI が `filePath` と行番号だけで文脈を復元できるようにする

## 対象ユーザー
- IntelliJ 系 IDE を使ってコードレビューを行う開発者
- AI にレビュー観点を引き継ぎたい開発者
- ローカルのファイルにコメントを残して、ブランチ単位で管理したい利用者

## コア機能
- 行または行範囲を選択すると、ガターに追加ボタンを表示する
- コメント入力後、インラインコメントとして表示する
- 既存コメントをクリックして再編集・削除できるようにする
- ブランチごとに `.inline-review-notes/<branch>.json` に保存する
- `resolvedAt` が入ったコメントは対応済みとして扱う

## データ方針
- 保存形式は JSON とする
- `filePath` はプロジェクトルートからの相対パスとする
- コメントの状態は `resolvedAt` の有無で表現する
- snippet や severity は初期実装では持たない

## 非機能要件
- 既存プロジェクトに影響を与えない
- `.gitignore` でローカル保存領域を除外する
- 行番号の追跡はブランチ切り替えや編集後にも破綻しにくくする

## 非対象
- 列単位の指定
- チーム共有機能
- プロジェクト識別子の追加
- severity の独立管理
- snippet / anchorText の保存

## 成功条件
- ユーザーが IDE 上で指摘を残せる
- AI が JSON だけで対象箇所と内容を把握できる
- ブランチごとのレビュー履歴をローカルで分離できる
